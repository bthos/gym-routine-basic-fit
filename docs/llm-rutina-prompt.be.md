# Стварэнне Руціны з дапамогай LLM

Гэта кіраўніцтва ператварае праграму трэніровак, якая існуе толькі ў вашай галаве
(ці ў нататках трэнера), у файл `rutina.json`. Трэба запоўніць кароткі спіс патрабаванняў,
уставіць гатовы prompt у **любы** LLM-чат (ChatGPT, Claude, Gemini — усё, што ёсць пад
рукой), і імпартаваць адказ у прыклад.

Веданне JSON не патрабуецца. Без уліковага запісу, без ключа API — гэта працуе ў
звычайным акне чата браузера.

Усе даведачныя даныя знаходзяцца ў публічным рэпазіторыі
[github.com/bthos/gym-routine-basic-fit](https://github.com/bthos/gym-routine-basic-fit).
Шаблон prompt паказвае LLM, дзе іх чытаць — вам **не** трэба самастойна капіраваць
схему ці файлы абсталявання.

| Даныя | URL |
|-------|-----|
| Схема (крыніца ісціны) | [rutina.schema.json](https://raw.githubusercontent.com/bthos/gym-routine-basic-fit/main/data/schema/rutina.schema.json) |
| Каталог абсталявання | [equipment.json](https://raw.githubusercontent.com/bthos/gym-routine-basic-fit/main/data/equipment.json) |
| Спіс залаў | [gyms.json](https://raw.githubusercontent.com/bthos/gym-routine-basic-fit/main/data/gyms.json) |
| Даведачны прыклад | [phase1-monday.json](https://raw.githubusercontent.com/bthos/gym-routine-basic-fit/main/data/examples/phase1-monday.json) |

> **LLM без доступу да інтэрнэту?** Спампуйце гэтыя чатыры файлы з рэпазіторыя і
> прыкладзіце іх да чата ўручную. Prompt ужо называе кожны файл.

## Як гэта працуе

```
1. Запоўніце спіс патрабаванняў (звычайны тэкст, ніжэй)
2. Скапіруйце шаблон prompt, устаўце свой спіс у REQUEST
3. Устаўце ўсё ў любы LLM-чат
4. Скапіруйце адказ JSON ад LLM
5. Імпартуйце ў PWA (устаўка ці загрузка .json)
      ✓ прайшло  → гатова
      ✗ не прайшло → скапіруйце памылкі валідацыі назад у LLM, папрасіце выправіць
                     і паўторна вывесці поўны JSON, вярніцеся да кроку 5
```

Цыкл выпраўлення ў кроку 5 — механізм надзейнасці. Паведамленні аб памылках напісаны
так, каб можна было ўставіць іх прама ў чат.

Лакальная валідацыя без прыкладу: `npm run validate-rutina -- шлях/да/rutina.json`

---

## Крок 1 — Запоўніце спіс патрабаванняў

Скапіруйце гэты блок, запоўніце кожную радок (звычайны тэкст, без JSON) і захоўвайце
яго для ўстаўкі ў секцыю `REQUEST` Кроку 2.

```text
1. Для каго гэта / назва праграмы:
2. Асноўная мэта ў гэтай фазе:
3. Дзён у тыдзень:
4. Бюджэт часу сесіі:
5. Траўмы / рухі, якіх трэба пазбягаць (напішыце "няма" калі няма):
6. Мэтавы трэнажорны зал (назва ці id — гл. gyms.json у DATA SOURCES):
7. Мова для выходнага тэксту:
8. Экспарт папярэдняга прагрэсу (неабавязкова — устаўце Markdown з экрана экспарту PWA або пакіньце пустым):
```

| # | Поле | Прыклад | Абавязковае |
|---|------|---------|-------------|
| 1 | Для каго гэта / назва праграмы | "Іван — Фаза 2" | так |
| 2 | Асноўная мэта ў гэтай фазе | "Гіпертрафія, больш аб'ёму" | так |
| 3 | Дзён у тыдзень | 4 | так |
| 4 | Бюджэт часу сесіі | "45-60 хвілін" | так |
| 5 | Траўмы / рухі, якіх трэба пазбягаць | "Пазбягаць ваеннага жыму праз правае плячо" | не — напішыце "няма" калі няма |
| 6 | Мэтавы трэнажорны зал | "Avda. Andalucía, Centro Comercial Alameda, Малага" | так — LLM фільтруе абсталяванне па id гэтага зала |
| 7 | Мова для выходнага тэксту | "Беларуская" | так |
| 8 | Экспарт папярэдняга прагрэсу | Markdown з экрана экспарту PWA | не — прапусціце для першай фазы |

## Крок 2 — Скапіруйце шаблон prompt

Гэта поўны шаблон, гатовы для ўстаўкі ў LLM-чат. Запоўніце адзін маркер у дужках
(`<<< ... >>>`) запоўненым спісам патрабаванняў з Кроку 1.

````text
{{PROMPT_TEMPLATE}}
````

Уключэнне "output only JSON" і ў пачатку (`ROLE`), і ў канцы (`OUTPUT`) — намерана;
гэта найбольш эфектыўная інструкцыя для атрымання разбіраемага выніку ад chat-LLM.

## Крок 3 — Імпарт у прыклад

1. Скапіруйце адказ JSON ад LLM (без markdown-агародж).
2. Адкрыйце экран **Імпарт** PWA і ўстаўце, ці загрузіце файл `.json`.
3. Націсніце **Імпарт**. Прыклад валідуе па той жа схеме і каталогу абсталявання.

Калі валідацыя не прайшла, скапіруйце спіс памылак з прыкладу і ўстаўце назад у
LLM-чат наступным паведамленнем. Папрасіце выправіць і паўторна вывесці поўны JSON.
Паўторыце, пакуль імпарт не ўдасціцца.

**Неабавязкова — валідацыя на камп'ютары:**

```bash
npm run validate-rutina -- шлях/да/вашай-rutina.json
```

| Вынік | Прыклад вываду | Што рабіць |
|---|---|---|
| Поспех | `✓ Valid rutina: 4 days, 22 exercises` | Імпарт у прыклад |
| Памылка схемы | `days[1].label: required` | Уставіць увесь блок памылак назад у LLM |
| Памылка ID абсталявання | `days[2].exercises[0].equipmentId "g3-xx" not found in data/equipment.json` | Тое ж — уставіць дослова |

## Практычны прыклад (адзін поўны цыкл)

**1 — Запоўнены спіс патрабаванняў:**

```text
1. Для каго гэта / назва праграмы: Назва — Фаза 2
2. Асноўная мэта ў гэтай фазе: Гіпертрафія, больш мышечнага аб'ёму
3. Дзён у тыдзень: 4
4. Бюджэт часу сесіі: 45-60 хвілін
5. Траўмы / рухі, якіх трэба пазбягаць: няма
6. Мэтавы трэнажорны зал: Avda. Andalucía, Centro Comercial Alameda (id 3)
7. Мова для выходнага тэксту: Беларуская
8. Экспарт папярэдняга прагрэсу (неабавязкова): (яшчэ няма, першы раз)
```

**2 — Адправлены prompt (спіс у REQUEST; LLM чытае схему і абсталяванне па URL):**

```text
### ROLE
You are a strength & conditioning coach. Output ONLY JSON...
### DATA SOURCES
https://raw.githubusercontent.com/bthos/gym-routine-basic-fit/main/data/schema/rutina.schema.json
...
### REQUEST
1. Для каго гэта / назва праграмы: Назва — Фаза 2
...
### OUTPUT
Return the JSON object now. Nothing else.
```

**3 — Чаканы вывад LLM (паказаны адзін дзень; рэальны адказ пакрывае усе 4 дні):**

```json
{
  "schemaVersion": 1,
  "program": {
    "name": "Назва — Фаза 2",
    "phaseName": "Гіпертрафія",
    "phaseNumber": 2,
    "gymId": 3,
    "durationWeeks": 12,
    "lastUpdated": "2026-07-13"
  },
  "phaseInfo": {
    "objective": "Максімальны мышечны аб'ём з правільнай тэхнікай",
    "intensityPercent": "70-75%",
    "restSeconds": "45-60",
    "frequencyPerWeek": 4
  },
  "days": [
    {
      "label": "Панядзелак",
      "exercises": [
        {
          "name": "Prensa de Pecho",
          "muscleGroups": ["pectoralis-major"],
          "equipmentId": "g3-s10",
          "sets": 4,
          "reps": "12-15",
          "restSeconds": 45,
          "intensity": "RIR 1-2",
          "technique": ["Спіна прыціснутая да спінкі"]
        }
      ]
    }
  ],
  "rules": ["Гідратацыя - 2-3 літры вады на дзень"],
  "notes": []
}
```

**4 — Імпарт** у PWA, ці `npm run validate-rutina -- data/rutina-nombre-fase2-draft.json`.

## Вырашэнне праблем

- **LLM не можа загрузіць URL.** Прыкладзіце чатыры файлы DATA SOURCES уручную, ці
  устаўце іх змесціва ў чат. Назвы файлаў супадаюць з указанымі ў prompt.
- **LLM дадаў уступ ці абгорнуў JSON у markdown.** Пераадправіце з: "Output ONLY the
  JSON object, no markdown fence, no explanation."
- **Адказ абрываецца сярод JSON (доўгія праграмы).** Папрасіце LLM "continue from
  where you stopped, still outputting only JSON".
- **ID абсталявання не знайдзены.** LLM павінен выбіраць id з `equipment.json`, дзе
  масіў `gyms` утрымлівае id вашага зала — не выдуманыя id. Уставіце памылку
  валідатара і папрасіце перачитаць каталог.
- **Не ведаеце id свайго зала?** LLM чытае `gyms.json`; таксама можна адкрыць яго ў
  рэпазіторыі па спасылцы вышэй.

## Абнаўленне праграмы

Калі праграма загружана, яе можна замяніць ці выдаліць на ўкладцы **Programa**:

1. Адкрыйце **Programa** і пракруціце да канца агляду.
2. Націсніце **Reemplazar programa**, каб перайсці на экран імпарту і ўставіць новы rutina.json.
3. Ці **Eliminar programa**, каб ачысціць актыўную праграму.

Гісторыя сесій захоўваецца пры выдаленні — запісы застаюцца, але больш не прывязаныя да праграмы.
