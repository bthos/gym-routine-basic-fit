# Стварэнне Руціны з дапамогай LLM

Гэта кіраўніцтва ператварае праграму трэніровак, якая існуе толькі ў вашай галаве
(ці ў нататках трэнера), у файл `rutina.json`. Запоўніце prompt у верхняй частцы гэтага
экрана, скапіруйце яго ў **любы** LLM-чат (ChatGPT, Claude, Gemini — усё, што ёсць пад
рукой), і імпартуйце адказ у прыклад **Rutina**.

Веданне JSON не патрабуецца. Без уліковага запісу, без ключа API — гэта працуе ў
звычайным акне чата браузера.

Усе даведачныя даныя знаходзяцца ў публічным рэпазіторыі
[github.com/bthos/gym-routine-basic-fit](https://github.com/bthos/gym-routine-basic-fit).
Prompt паказвае LLM, дзе іх чытаць — вам **не** трэба самастойна капіраваць схему ці
файлы абсталявання.

| Даныя | URL |
|-------|-----|
| Схема (крыніца ісціны) | [rutina.schema.json](https://raw.githubusercontent.com/bthos/gym-routine-basic-fit/main/data/schema/rutina.schema.json) |
| Каталог абсталявання | [equipment.json](https://raw.githubusercontent.com/bthos/gym-routine-basic-fit/main/data/equipment.json) |
| Спіс залаў (для LLM) | [gyms.json](https://raw.githubusercontent.com/bthos/gym-routine-basic-fit/main/data/gyms.json) |
| Даведачны прыклад | [phase1-monday.json](https://raw.githubusercontent.com/bthos/gym-routine-basic-fit/main/data/examples/phase1-monday.json) |

> **LLM без доступу да інтэрнэту?** Спампуйце чатыры файлы даных з рэпазіторыя і
> прыкладзіце іх да чата ўручную. Prompt ужо называе кожны файл.

## Як гэта працуе

```
1. Запоўніце секцыю REQUEST на пачатку prompt (блок са спісам залаў — над ім)
2. Скапіруйце ўвесь prompt і ўстаўце ў любы LLM-чат
3. Скапіруйце адказ JSON ад LLM
4. Імпартуйце ў прыклад Rutina (устаўка ці загрузка .json)
      ✓ прайшло  → гатова
      ✗ не прайшло → скапіруйце памылкі валідацыі назад у LLM, папрасіце выправіць
                     і паўторна вывесці поўны JSON, вярніцеся да кроку 4
```

Цыкл выпраўлення ў кроку 4 — механізм надзейнасці. Паведамленні аб памылках напісаны
так, каб можна было ўставіць іх прама ў чат.

Лакальная валідацыя без прыкладу: `npm run validate-rutina -- шлях/да/rutina.json`

## Крок 1 — Скапіруйце prompt

Скарыстайцеся кнопкай **Скапіраваць** над гэтым кіраўніцтвам. Prompt пачынаецца з
**REQUEST** — запоўніце кожны нумараваны радок (звычайны тэкст, без JSON) перад адпраўкай.

> **Поле 6 — мэтавы зал**
> Знайдзіце назву і лічбавы **id** вашага зала ў **[спісе залаў →](https://cdn.jsdelivr.net/gh/bthos/gym-routine-basic-fit@main/gyms.html)**
> (адкрываецца ў новай укладцы). Таксама на ўкладцы **Catálogo** у прыкладзе Rutina.

| # | Поле | Прыклад | Абавязковае |
|---|------|---------|-------------|
| 1 | Для каго гэта / назва праграмы | "Іван — Фаза 2" | так |
| 2 | Асноўная мэта ў гэтай фазе | "Гіпертрафія, больш аб'ёму" | так |
| 3 | Дзён у тыдзень | 4 | так |
| 4 | Бюджэт часу сесіі | "45-60 хвілін" | так |
| 5 | Траўмы / рухі, якіх трэба пазбягаць | "Пазбягаць ваеннага жыму праз правае плячо" | не — напішыце "няма" калі няма |
| 6 | Мэтавы трэнажорны зал | "Avda. Andalucía, Centro Comercial Alameda (id 3)" | так |
| 7 | Мова для выходнага тэксту | "Беларуская" | так |
| 8 | Экспарт папярэдняга прагрэсу | Markdown з **Historial → Exportar** у Rutina | не — прапусціце для першай фазы |

Уключэнне "output only JSON" і ў пачатку (`ROLE`), і ў канцы (`OUTPUT`) — намерана.

## Крок 2 — Імпарт у прыклад Rutina

1. Скапіруйце адказ JSON ад LLM (без markdown-агародж).
2. Адкрыйце экран **Імпарт** у прыкладзе Rutina і ўстаўце, ці загрузіце файл `.json`.
3. Націсніце **Імпарт**. Прыклад валідуе па той жа схеме і каталогу абсталявання.

Калі валідацыя не прайшла, скапіруйце спіс памылак і ўстаўце назад у LLM-чат.
Папрасіце выправіць і паўторна вывесці поўны JSON. Паўторыце, пакуль імпарт не ўдасціцца.

**Неабавязкова — валідацыя на камп'ютары:**

```bash
npm run validate-rutina -- шлях/да/вашай-rutina.json
```

| Вынік | Прыклад вываду | Што рабіць |
|---|---|---|
| Поспех | `✓ Valid rutina: 4 days, 22 exercises` | Імпарт у прыклад |
| Памылка схемы | `days[1].label: required` | Уставіць увесь блок памылак назад у LLM |
| Памылка ID абсталявання | `days[2].exercises[0].equipmentId "g3-xx" not found` | Тое ж — уставіць дослова |

## Практычны прыклад

**REQUEST запоўнены (верх скапіраванага prompt):**

```text
1. Для каго гэта / назва праграмы: Назва — Фаза 2
2. Асноўная мэта ў гэтай фазе: Гіпертрафія, больш мышечнага аб'ёму
3. Дзён у тыдзень: 4
4. Бюджэт часу сесіі: 45-60 хвілін
5. Траўмы / рухі, якіх трэба пазбягаць: няма
6. Мэтавы трэнажорны зал: Avda. Andalucía, Centro Comercial Alameda (id 3)
7. Мова для выходнага тэксту: Беларуская
8. Экспарт папярэдняга прагрэсу (неабавязкова): (яшчэ няма)
```

**LLM чытае схему і абсталяванне па URL у DATA SOURCES і вяртае JSON.**

**Імпарт** у Rutina, ці `npm run validate-rutina -- data/rutina-nombre-fase2-draft.json`.

## Вырашэнне праблем

- **LLM не можа загрузіць URL.** Прыкладзіце чатыры файлы DATA SOURCES уручную.
- **LLM дадаў уступ ці абгорнуў JSON у markdown.** "Output ONLY the JSON object, no markdown fence, no explanation."
- **Адказ абрываецца сярод JSON.** "continue from where you stopped, still outputting only JSON".
- **ID абсталявання не знайдзены.** LLM павінен выбіраць id з `equipment.json` для вашага зала.
- **Не ведаеце id зала?** [Спіс залаў](https://cdn.jsdelivr.net/gh/bthos/gym-routine-basic-fit@main/gyms.html) ці ўкладка **Catálogo** у Rutina.
